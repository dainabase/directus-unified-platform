# Migration Guide: v0.1.x to v0.2.0

## Overview
Version 0.2.0 is a major expansion of the Design System, adding 13 new components and improving the overall architecture.

## Breaking Changes

### Import Changes
```typescript
// Before (v0.1.x)
import { Button } from '@dainabase/ui/components';

// After (v0.2.0)
import { Button } from '@dainabase/ui';
```

### Theme Provider Required
All components now require ThemeProvider at the root:

```typescript
// App.tsx
import { ThemeProvider } from '@dainabase/ui';

function App() {
  return (
    <ThemeProvider>
      {/* Your app */}
    </ThemeProvider>
  );
}
```

## New Features

### New Components
- CommandPalette
- DataGrid & DataGridAdv
- DatePicker
- Dialog & Sheet
- Charts
- Form, Checkbox, Select, Switch
- ThemeToggle

### Dark Mode Support
All components now support dark mode automatically when wrapped in ThemeProvider.

## Migration Steps

1. **Update package**
```bash
pnpm update @dainabase/ui@0.2.0
```

2. **Add ThemeProvider**
Wrap your app with ThemeProvider.

3. **Update imports**
Change all component imports to use the barrel export.

4. **Test thoroughly**
Especially test dark mode and new TypeScript types.

## Deprecated Features
- Direct component imports from subpaths
- Manual dark mode handling

## Support
For migration assistance, please open an issue on GitHub.
