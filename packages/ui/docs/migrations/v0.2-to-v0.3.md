# Migration Guide: v0.2.x to v0.3.0

## Overview
Version 0.3.0 adds advanced date handling components and improves test coverage.

## New Components
- Calendar (with multiple selection modes)
- DateRangePicker
- Popover (utility component)

## Non-Breaking Changes
This release contains no breaking changes. All existing components remain compatible.

## New Features

### Calendar Component
```typescript
import { Calendar } from '@dainabase/ui';

// Single date selection
<Calendar mode="single" />

// Multiple dates
<Calendar mode="multiple" />

// Date range
<Calendar mode="range" />
```

### DateRangePicker
```typescript
import { DateRangePicker } from '@dainabase/ui';

<DateRangePicker
  onSelect={(range) => console.log(range)}
  presets={[
    { label: 'Last 7 days', days: 7 },
    { label: 'Last 30 days', days: 30 }
  ]}
/>
```

## Migration Steps

1. **Update package**
```bash
pnpm update @dainabase/ui@0.3.0
```

2. **No code changes required**
Existing code will continue to work.

3. **Optional: Adopt new components**
Replace custom date pickers with the new components.

## Improvements
- Test coverage increased to ~80%
- Better TypeScript types
- Improved accessibility
